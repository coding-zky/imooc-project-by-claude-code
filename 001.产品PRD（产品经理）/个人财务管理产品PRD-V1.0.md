## 一、产品概述

### 1.1 产品背景

现代人日常生活中面临繁杂的消费场景，从日常餐饮、交通出行到购物娱乐，每一笔支出都值得关注。但大多数人在消费后缺乏系统性的记录习惯，导致月末回顾时对资金流向模糊不清，难以进行合理的财务规划。

本产品旨在帮助用户**便捷地记录每一笔消费**，通过清晰的分类统计和可视化呈现，让用户对个人财务状况一目了然，从而培养良好的消费习惯，实现理性消费与财务健康。

### 1.2 产品定位

**产品名称：** 轻账本（LightLedger）

**产品类型：** 个人财务记录工具（Web 端）

**核心定位：** 极简、快速、无负担的个人消费记录工具

**目标用户：**

- 有日常记账需求但厌恶复杂操作的用户
- 希望了解个人消费结构的学生、职场人士
- 追求简洁工具而非臃肿功能的技术型用户

### 1.3 第一版范围说明

| 维度     | 说明                                  |
| -------- | ------------------------------------- |
| 端侧     | Web 网页端                            |
| 功能     | 消费记录、分类统计、数据可视化        |
| 存储     | 本地浏览器存储（LocalStorage）        |
| 账号体系 | 用户名 + 密码注册登录（暂不集成 JWT） |
| 后端     | 第一版无需后端，纯前端实现            |

**明确排除（后续版本迭代）：**

- 移动端 App
- 小程序
- JWT 鉴权
- 云端数据同步
- 多人协作

## 二、用户故事

### 2.1 核心用户故事

| 角色         | 场景           | 诉求                                      |
| ------------ | -------------- | ----------------------------------------- |
| 作为普通用户 | 每天消费后     | 希望 30 秒内完成记录，操作步骤不超过 3 步 |
| 作为普通用户 | 月底时         | 希望一目了然地看到各分类的消费占比        |
| 作为普通用户 | 回看历史记录时 | 希望快速找到特定日期或分类的消费明细      |

| 作为回归用户 | 重新访问时     | 希望直接进入记录页面，而非再次登录引导    |

### 3.2 用户模块

#### 3.2.1 用户注册

**功能描述：** 新用户通过填写用户名和密码创建账号。

**详细规则：**

| 字段     | 规则                                              |
| -------- | ------------------------------------------------- |
| 用户名   | 4-20 个字符，支持字母、数字、下划线；**必须唯一** |
| 密码     | 最少 6 个字符                                     |
| 确认密码 | 必须与密码一致                                    |

**交互流程：**

```
[打开注册页] → [填写表单] → [点击注册] → [系统校验] → [注册成功跳转首页]
                                                    ↓
                                            [注册失败 → 显示错误提示]
```

**错误提示文案：**

| 场景           | 提示文案                             |
| -------------- | ------------------------------------ |
| 用户名已存在   | "该用户名已被注册，请尝试其他用户名" |
| 用户名为空     | "请输入用户名"                       |
| 密码过短       | "密码至少需要 6 个字符"              |
| 两次密码不一致 | "两次输入的密码不一致"               |
| 用户名格式错误 | "用户名只能包含字母、数字和下划线"   |

**页面路径：** `/register`

#### 3.2.2 用户登录

**功能描述：** 已注册用户通过用户名和密码登录。

**交互流程：**

```
[打开登录页] → [填写表单] → [点击登录] → [系统校验] → [登录成功跳转首页/仪表盘]
                                                    ↓
                                            [登录失败 → 显示错误提示]
```

**错误提示文案：**

| 场景         | 提示文案                 |
| ------------ | ------------------------ |
| 用户名不存在 | "用户名不存在，请先注册" |
| 密码错误     | "密码错误，请重新输入"   |
| 用户名为空   | "请输入用户名"           |
| 密码为空     | "请输入密码"             |

**页面路径：** `/login`

#### 3.2.3 用户登出

**功能描述：** 已登录用户退出当前账号。

**交互流程：**

```
[点击登出] → [清除登录状态] → [跳转登录页]
```

**全局拦截：** 未登录状态下访问任何需要登录的页面，均自动跳转至登录页。

#### 3.2.4 登录状态管理

- 使用 `localStorage` 存储当前登录用户信息
- 存储结构：`{ username: string, loginAt: timestamp }`
- 登录状态有效期：浏览器关闭前有效（会话级）

### 3.3 首页 / 仪表盘

**功能描述：** 用户登录后的主页面，展示当月消费概况。

**页面路径：** `/` （已登录用户默认跳转此处）

**核心信息展示：**

| 模块            | 内容                                              |
| --------------- | ------------------------------------------------- |
| 本月总消费      | 当月所有消费记录的总金额，单位：元，保留 2 位小数 |
| 今日消费        | 当天所有消费记录的总金额                          |
| 消费分类饼图    | 各分类消费金额占比（交互待定，可用图表库实现）    |
| 近 7 日消费趋势 | 折线图展示近 7 天每日消费金额                     |
| 快速记录入口    | 一键跳转至记录消费页面                            |

**空状态设计：**

- 若本月尚无任何消费记录，显示引导插画 + 文案："还没有消费记录，开始记录你的第一笔支出吧"
- 提供「立即记录」按钮

### 3.4 记录消费

**功能描述：** 用户添加一条新的消费记录。

**页面路径：** `/add`

**表单字段：**

| 字段     | 类型     | 必填 | 说明                         |
| -------- | -------- | ---- | ---------------------------- |
| 消费金额 | 数字输入 | 是   | 正数，支持到分（2 位小数）   |
| 消费分类 | 下拉选择 | 是   | 预设分类列表（见 3.4.1）     |
| 消费日期 | 日期选择 | 是   | 默认为当天，支持选择历史日期 |
| 备注     | 文本输入 | 否   | 最长 200 字符，用于补充说明  |

#### 3.4.1 预设消费分类

| 分类名称 | 图标（emoji） | 说明                           |
| -------- | ------------- | ------------------------------ |
| 餐饮     | 🍜            | 早中晚餐、零食、饮料等         |
| 交通     | 🚗            | 公交、地铁、打车、油费、停车等 |
| 购物     | 🛒            | 服饰、日用品、数码产品等       |
| 娱乐     | 🎮            | 游戏、电影、音乐、娱乐活动等   |
| 居住     | 🏠            | 房租、水电燃气、物业等         |
| 医疗     | 💊            | 医院、药店、保健品等           |
| 教育     | 📚            | 书籍、课程、培训等             |
| 通讯     | 📱            | 手机话费、网络套餐等           |
| 社交     | 🎁            | 送礼、人情往来等               |
| 其他     | 📌            | 无法归类的支出                 |

**交互规则：**

- 金额输入框自动获得焦点
- 输入金额后按回车可直接提交（快捷操作）
- 提交成功后显示成功提示（toast），并清空表单，可继续添加
- 提交成功后 1.5 秒自动跳转回首页/仪表盘

**错误处理：**

| 场景       | 提示                 |
| ---------- | -------------------- |
| 金额为空   | "请输入消费金额"     |
| 金额 <= 0  | "消费金额必须大于 0" |
| 分类未选择 | "请选择消费分类"     |

### 3.5 消费明细

**功能描述：** 查看历史消费记录列表，支持筛选和搜索。

**页面路径：** `/records`

**列表展示：**

| 字段 | 说明                               |
| ---- | ---------------------------------- |
| 日期 | 显示格式：YYYY-MM-DD（星期X）      |
| 分类 | 显示分类图标 + 名称                |
| 金额 | 显示金额，单位：元                 |
| 备注 | 如有备注，显示在金额下方，灰色小字 |

**筛选功能：**

| 筛选维度   | 实现方式                     |
| ---------- | ---------------------------- |
| 按月份筛选 | 月份选择器，默认当月         |
| 按分类筛选 | 分类下拉框，支持「全部分类」 |

**排序规则：**

- 默认按日期倒序（最新在前）
- 支持切换为正序

**分页设计：**

- 每页显示 20 条记录
- 底部加载更多（无限滚动或分页按钮均可）

**空状态：**

- 无任何记录时显示："该筛选条件下没有消费记录"

**交互操作：**

- 点击单条记录可展开/收起详情（仅展示，不支持编辑）
- 支持删除操作（需二次确认）

### 3.6 统计报表

**功能描述：** 展示消费数据的统计分析。

**页面路径：** `/stats`

**统计维度：**

| 维度                 | 可视化方式        |
| -------------------- | ----------------- |
| 各分类消费占比       | 饼图 / 环形图     |
| 近 30 天每日消费趋势 | 折线图            |
| 各分类消费金额排名   | 横向柱状图 / 列表 |

**时间范围选择：**

- 默认展示当月数据
- 支持选择近 7 天、近 30 天、自定义日期范围

**数据说明：**

- 图表旁边显示具体数字
- 若某分类金额为 0，不显示在图表中（避免干扰）

## 四、非功能需求

### 4.1 性能需求

| 指标         | 要求                                      |
| ------------ | ----------------------------------------- |
| 首屏加载时间 | ≤ 2 秒（网络正常情况下）                  |
| 操作响应时间 | ≤ 200ms（除网络请求外）                   |
| 数据存储上限 | 单用户数据量预估 < 10MB（本地存储限制内） |

### 4.2 兼容性需求

| 项目       | 要求                                                                  |
| ---------- | --------------------------------------------------------------------- |
| 浏览器支持 | Chrome（最新版）、Firefox（最新版）、Safari（最新版）、Edge（最新版） |
| 屏幕适配   | 桌面端优先，适配 1280px 及以上宽度                                    |
| 响应式布局 | 最小支持宽度 375px（移动端预览）                                      |

### 4.3 数据存储方案

**技术选型：** `localStorage`

**数据模型：**

```json
// 用户信息（key: users）
{
  "users": [
    { "username": "string", "password": "string（可加密存储）", "createdAt": "timestamp" }
  ]
}

// 消费记录（key: records_{username}）
{
  "records": [
    {
      "id": "uuid",
      "amount": 0.00,
      "category": "string",
      "date": "YYYY-MM-DD",
      "note": "string",
      "createdAt": "timestamp"
    }
  ]
}
```

**数据隔离：** 不同用户的数据完全隔离，通过用户名作为 key 前缀区分。

### 4.4 安全考虑

| 措施     | 说明                                                                       |
| -------- | -------------------------------------------------------------------------- |
| 密码存储 | 虽是第一版，仍建议对密码进行简单加密（如 base64 或简易哈希），避免明文存储 |
| 防止 XSS | 所有用户输入内容在渲染时需做转义处理                                       |
| 表单校验 | 前端对所有输入进行校验，不信任任何客户端数据                               |

**设计理念：** 简洁、克制、高效

### 7.3 数据存储封装

```javascript
// utils/storage.js 示例

const STORAGE_KEYS = {
  USERS: "lightledger_users",
  RECORDS_PREFIX: "lightledger_records_",
};

export const storage = {
  get(key) {
    const data = localStorage.getItem(key);
    return data ? JSON.parse(data) : null;
  },

  set(key, value) {
    localStorage.setItem(key, JSON.stringify(value));
  },

  remove(key) {
    localStorage.removeItem(key);
  },

  getUserRecords(username) {
    return this.get(STORAGE_KEYS.RECORDS_PREFIX + username) ?? { records: [] };
  },

  saveUserRecords(username, records) {
    this.set(STORAGE_KEYS.RECORDS_PREFIX + username, { records });
  },
};
```
